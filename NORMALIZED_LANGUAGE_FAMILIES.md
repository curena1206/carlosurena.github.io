# Normalized Language Families

_Active as of 2026-05-15. Apply to any new content, edit, or review across the CarlosUrena.com ecosystem._

---

## How to use this document

When editing or creating any content for the site, PFI, models, or consulting materials:

1. Check the output against every family below.
2. If a term from any family appears in user-visible output, re-open only that string for review.
3. Apply the accepted replacement or leave unchanged only if it qualifies as an accepted exception.
4. Frozen content is not exempt. If a frozen string contains a family term, surface it for a targeted decision.

---

## OPTIMIZATION FAMILY

**Members:**
- optimize
- optimized
- optimizing
- optimization
- optimizations
- optimization program
- optimize returns
- total returns optimization
- routing and cost optimization
- optimizing for scale

**Accepted exceptions (do not flag):**
- routing optimization in technical context (e.g. 'rail routing optimization rate' as a KPI metric)
- optimization in passive/descriptive context describing a mathematical model output
- 'calibrated' is the approved alternative for descriptive optimization language

**Approved replacements:** improve / extend / advance / strengthen / calibrate / deepen

## How to apply this going forward

Before writing or editing any content:

1. Draft the content.
2. Check each sentence against the family member lists above.
3. If a match is found: apply the approved replacement or confirm it qualifies as an accepted exception.
4. If working with frozen PFI content: surface any family match for a targeted single-string decision.
5. Do not run broad audits. One string at a time, only when triggered by a family match.
